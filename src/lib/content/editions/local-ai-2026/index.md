The local AI market has split into four practical layers: model checkpoints, inference runtimes, model managers, and agent or assistant environments. Productive local systems are assembled from these layers rather than bought as a single universal stack.

The major trade-off is no longer whether local AI is viable. It is which work belongs on which machine. Small and mid-sized coding models are now credible on Apple Silicon and laptop-class CUDA systems. Large sparse MoE models remain workstation or server workloads when latency, context length and reliability matter.

This edition treats benchmark claims as evidence, not as verdicts. Vendor-reported scores identify capability direction. Community reproductions and local measurements remain necessary before operational commitment. Hardware feasibility rows are estimates unless labeled otherwise.

Hermes Agent is used as a named comparison anchor because it represents the local-first autonomous assistant direction: tools, memory, loops, routines and user-controlled execution. It is not treated as the default winner.
